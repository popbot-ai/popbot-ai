# Фазы

Дорожная карта перехода PopBot от «дизайн + прототип» к «полезный ежедневный инструмент». Отражает разбиение на фазы в [POPBOT_DESIGN.md](POPBOT_DESIGN.md#phasing), но отслеживает конкретный прогресс с помощью чекбоксов.

Обновляйте этот файл по мере продвижения. Один коммит может отмечать несколько пунктов.

---

## Фаза 0 — Предпосылки (~3 дня)

Основополагающие элементы в репозитории AutoRPG + нативный помощник здесь. Большинство из них блокируют реальное сквозное тестирование, но не каркас Electron.

### В `~/pop/autorpg`

- [ ] **Переопределение env `POPBOT_MCP_PORT`** для MCP-сервера внутри редактора (`autorpg-unity/Assets/Editor/MCP/UnityMcpServer.cs`). Читать порт из env, откатываться к `17893`. ~5 мин.
- [ ] **Флаги `./run_local.sh --port` + `--data-dir`.** Сервер принимает оба как аргументы; каталог данных для изоляции БД на каждый слот. ~30 мин.
- [ ] **Расширение эндпоинта `/health`** — возвращать `{ ok, commit, gameDataHash, dtoVersion, uptimeSec }`. PopBot использует это для обнаружения рассинхронизации во время аренды. ~30 мин.

### В этом репозитории

- [ ] **Нативный помощник перемещения окон для macOS** — Swift CLI в `native/popbot-windowmover/`. Подкоманды: `move`, `minimize`, `wait-for-window`. ~½ дня.
- [ ] **Прототип жизненного цикла слота** — отдельный TS-модуль в `src/main/slots/`, прогоняемый скриптом в `scripts/`. Охватывает worktree add, Library COW от master, переключение ветки с безопасностью stash, аренду/освобождение, сверку сирот. ~1 день.

## Фаза 1 — Поверхность автоматизации MCP (~3-5 дней)

В `~/pop/autorpg`. Строит инструменты MCP внутри редактора, которые агенты будут реально использовать.

- [ ] **Инфраструктура задач** — `job_status`, `job_get_result`, `job_cancel`, `job_list`. Все долгие инструменты немедленно возвращают `{ jobId }`.
- [ ] **Инструменты жизненного цикла** — `play_status`, `play_enter` (job), `play_exit`, `play_pause/resume/step`, `time_scale_set`, `editor_quit`.
- [ ] **Инструменты наблюдения** — `screenshot`, `game_state_summary`, `screen_stack`, `chapter_status`, `ui_tree`, `ui_query`.
- [ ] **Инструменты действия** — `ui_click`, `ui_click_by_loc`.
- [ ] **Инструменты синхронизации** — `wait_until` (job), `wait_for_idle` (job).
- [ ] **Инструменты логов / сервера** — расширенный `console_get_logs` (`sinceTimestamp`, `dedupe`, `dumpTo`, `includeStack`), `server_logs`, `server_health`, `client_set_server_endpoint`.
- [ ] **Сессии** — `mcp_session_start`, `mcp_session_end` для предсказуемых директорий артефактов.
- [ ] **Миграция существующих долгих инструментов** на модель job: `rebuild_gamedata`, `rebuild_dtos`, `addressables_build`, `addressables_clean`.

## Фаза 2 — MVP PopBot Electron (~1-2 недели)

Пригодно для сквозного использования в одном чате. **В процессе.**

- [ ] **Каркас Electron** — `package.json`, Vite + React + TS + Tailwind, electron-builder, ESLint + Prettier, Vitest.
- [ ] **Разделение main / preload / renderer** с типизированным мостом IPC.
- [ ] **Перенос 8 прототипных JSX** в `.tsx` в `src/renderer/`. Статический UI работает в окне Electron без функциональной поддержки.
- [ ] **Схема better-sqlite3** — chats, messages, slots, prefs.
- [ ] **Одна сессия ClaudeBackend**, подключённая к одной колонке чата. Отправка сообщения, получение потока событий.
- [ ] **Движок политики `canUseTool`** — жёстко закодированный deny-список + разрешение по режиму. Renderer отображает запросы прав как модальные окна.
- [ ] **Подключён менеджер слотов** — один слот, реальный worktree, реальный запуск Unity через помощник Фазы 0.
- [ ] **Интеграция нативного помощника перемещения окон** — Unity открывается, помощник размещает его на экране 2.
- [ ] **Скелет панели настроек** — режим для каждого чата, режим сервера, масштаб времени, бэкенд агента.
- [ ] **Демо сквозного цикла** — открыть чат → агент читает код → агент запускает игру → агент делает скриншоты → агент отчитывается.

## Фаза 3 — Мультичат + панели очереди внимания (~1-2 недели)

Задействует [US-1](USER_STORIES.md#us-1--осведомлённость-об-очереди-внимания), [US-2](USER_STORIES.md#us-2--активация-в-один-клик), [US-5](USER_STORIES.md#us-5--лёгкая-многозадачность-через-превью), [US-6](USER_STORIES.md#us-6--статус-с-первого-взгляда).

- [ ] Несколько колонок чата; плавающее добавление/удаление.
- [ ] Полоса миниатюр с цветовыми статусами (US-5, US-6).
- [ ] **Панель тикетов Linear** (назначенные на меня, ранжированные по приоритету + сроку).
- [ ] **Панель непроверенных PR** (`gh` GraphQL).
- [ ] **Панель Slack** — личные сообщения, @упоминания, каналы во владении. Совершенно новая подсистема (`src/main/slack/`); OAuth через `keytar`. См. [USER_STORIES.md → Отклонения](USER_STORIES.md#slack-как-третий-источник-внимания-us-1).
- [ ] **Создание чата в один клик** из любой строки панели; чат засеян контекстом источника (US-2).
- [ ] Нижняя панель логов — вкладки Unity + сервер, синхронная прокрутка для активного чата.
- [ ] Переключатели mode + server-mode в настройках чата, с перенаправлением в середине сессии.
- [ ] Обнаружение рассинхронизации при аренде `remote-dev`.

## Фаза 4 — Полировка + продвинутое

- [ ] **Адаптер бэкенда Codex** — `CodexBackend implements AgentBackend`, возможности отмечены флагами в UI.
- [ ] **Headless `Window Mode`** — opt-in после того, как скрипт валидации batchmode докажет, что это работает на AutoRPG.
- [ ] Инструменты MCP **`crash_dump`, `events_pop`, `command_apply`, управление фикстурами**.
- [ ] **Корреляция логов по времени бок о бок** между панелями Unity и сервера.
- [ ] Уточнение **бюджетов автономности + обнаружения циклов** (триггеры паузы по токенам / времени / повторяющимся сбоям).
- [ ] **Канал обновлений** — автообновление через electron-builder + подписанные сборки.

## Открытые вопросы (перенесённые из дизайна)

1. Действительно ли AutoRPG работает в режиме Play с `-batchmode`? Скрипт валидации примерно в Фазе 4; не блокирует v1.
2. Периодичность обновления Master Library — ручная кнопка vs авто vs TTL в N дней? По умолчанию: ручная кнопка в prefs.
3. Количество слотов по умолчанию — жёстко заданные 4, или масштабирование по RAM/ядрам? Вероятно, по умолчанию 2-3, настраивается.
